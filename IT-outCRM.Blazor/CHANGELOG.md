# 📝 История изменений IT-outCRM

## 🎉 Версия 1.1 - Система ролей (2025-11-23)

### ✨ Новые функции

#### 🎭 Система ролей пользователей
- Добавлена поддержка трех ролей: **Администратор**, **Менеджер**, **Клиент**
- Автоматическая адаптация интерфейса под роль пользователя
- Условное отображение элементов навигации и контента

#### 📄 Новые страницы

1. **EditOrder** (`/orders/edit/{id}`)
   - Полнофункциональное редактирование заказов
   - Выбор клиента, исполнителя и статуса из списков
   - Валидация всех полей
   - Красивый UI с разделами

2. **EditCustomer** (`/customers/edit/{id}`)
   - Редактирование информации о клиенте
   - Выбор аккаунта и компании
   - Валидация и обработка ошибок

3. **Profile** (`/profile`)
   - Личный профиль пользователя
   - Отображение роли и прав доступа
   - Информация о безопасности
   - Аватар с инициалами
   - Цветная индикация роли (красная - админ, желтая - менеджер, синяя - клиент)

4. **MyOrders** (`/my-orders`)
   - Персональная страница заказов для клиентов
   - Статистика по заказам (всего, в работе, завершено, сумма)
   - Поиск по названию и описанию
   - Фильтрация по статусам
   - Card-based дизайн

#### 🔧 Улучшения существующих страниц

**Dashboard:**
- Теперь адаптируется под роль пользователя
- Для админов и менеджеров - полная статистика
- Для клиентов - персональная статистика по их заказам

**NavMenu:**
- Динамическое меню в зависимости от роли
- Отображение имени пользователя и роли
- Условное отображение пунктов меню:
  - Админы видят: Dashboard, Все заказы, Клиенты, Отчеты, Профиль
  - Менеджеры видят: Dashboard, Все заказы, Клиенты, Профиль
  - Клиенты видят: Dashboard, Мои заказы, Профиль

### 🎨 Дизайн и UX

- Современный градиентный дизайн
- Анимации при наведении на карточки
- Цветовая индикация ролей и статусов

## 🎯 Версия 1.0 - MVP (2025-11-22)

### ✨ Первоначальный релиз

#### Основные функции
- ✅ Аутентификация пользователей (Login/Register)
- ✅ JWT-токены для авторизации
- ✅ Dashboard с базовой статистикой
- ✅ Управление заказами (CRUD)
- ✅ Управление клиентами (CRUD)
- ✅ Просмотр деталей заказов и клиентов
- ✅ Пагинация списков
- ✅ Поиск и фильтрация

#### Реализованные страницы
- `/` - Главная (landing page)
- `/login` - Вход
- `/register` - Регистрация
- `/dashboard` - Панель управления
- `/orders` - Список заказов
- `/orders/create` - Создание заказа
- `/orders/{id}` - Детали заказа
- `/customers` - Список клиентов
- `/customers/create` - Создание клиента
- `/customers/{id}` - Детали клиента

#### Архитектура
- **Frontend:** Blazor Server (.NET 10)
- **Backend API:** ASP.NET Core Web API
- **Аутентификация:** JWT Bearer Token
- **Storage:** `TokenStorage` с in-memory cache
- **State Management:** Custom `AuthenticationStateProvider`

#### Сервисы
- `IAuthService` - аутентификация
- `IOrderService` - заказы
- `ICustomerService` - клиенты
- `IExecutorService` - исполнители
- `ICompanyService` - компании
- `IAccountService` - аккаунты
- `IAccountStatusService` - статусы аккаунтов
- `IOrderStatusService` - статусы заказов

#### UI/UX
- Современный Material Design
- Bootstrap 5 + Bootstrap Icons
- Адаптивный дизайн
- Анимации и переходы
- Градиенты и тени для глубины

### 🐛 Исправленные проблемы в MVP

1. **FormName requirement** (.NET 8+)
   - Добавлен атрибут `FormName` ко всем `EditForm`
   - Добавлен `[SupplyParameterFromForm]` к моделям

2. **AuthenticationHttpClientHandler**
   - Исправлена проблема с применением к `IAuthService`
   - Login/Register теперь не требуют токена

3. **Prerendering issues**
   - Отключен prerendering для интерактивных компонентов
   - Исправлена работа с `ProtectedLocalStorage`

4. **Navigation после login**
   - Исправлен редирект после успешного входа
   - Обновление меню после изменения состояния аутентификации

5. **Token persistence**
   - Реализован in-memory cache + localStorage
   - Токен сохраняется между перезагрузками страницы
