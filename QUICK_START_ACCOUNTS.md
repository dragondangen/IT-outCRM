# 🚀 Быстрый старт: Создание аккаунтов

## 📋 Порядок действий

### Шаг 1: Запустите приложение

```bash
cd IT-outCRM
dotnet run
```

Приложение запустится на `http://localhost:5295`

---

### Шаг 2: Откройте Swagger UI

Перейдите в браузере:
```
http://localhost:5295/swagger
```

---

### Шаг 3: Авторизуйтесь

#### 3.1. Войдите как администратор:

1. Разверните секцию **Auth** → `POST /api/auth/login`
2. Нажмите **"Try it out"**
3. Введите данные:
```json
{
  "username": "admin",
  "password": "Admin123!"
}
```
4. Нажмите **"Execute"**
5. Скопируйте **токен** из ответа (поле `token`)

#### 3.2. Настройте авторизацию в Swagger:

1. Нажмите кнопку **"Authorize"** 🔓 (в правом верхнем углу)
2. В поле введите: `Bearer ваш_скопированный_токен`
3. Нажмите **"Authorize"**
4. Нажмите **"Close"**

✅ Теперь вы авторизованы как администратор!

---

### Шаг 4: Проверьте существующие статусы

1. Разверните секцию **AccountStatuses**
2. Найдите `GET /api/accountstatuses`
3. Нажмите **"Try it out"**
4. Нажмите **"Execute"**

**Вы увидите список статусов:**
```json
[
  {
    "id": "...",
    "name": "Активный"
  },
  {
    "id": "...",
    "name": "Неактивный"
  },
  {
    "id": "...",
    "name": "Потенциальный"
  },
  {
    "id": "...",
    "name": "Заблокирован"
  }
]
```

📝 **Важно:** Скопируйте **ID** нужного статуса (например, "Активный")

---

### Шаг 5 (опционально): Создайте новый статус

Если нужного статуса нет:

1. Разверните секцию **AccountStatuses**
2. Найдите `POST /api/accountstatuses`
3. Нажмите **"Try it out"**
4. Введите данные:
```json
{
  "name": "Тестовый"
}
```
5. Нажмите **"Execute"**
6. Скопируйте **ID** созданного статуса из ответа

---

### Шаг 6: Создайте аккаунт

1. Разверните секцию **Accounts**
2. Найдите `POST /api/accounts`
3. Нажмите **"Try it out"**
4. Введите данные:
```json
{
  "companyName": "ООО Технологии",
  "foundingDate": "2020-01-15T00:00:00Z",
  "accountStatusId": "вставьте_ID_статуса_сюда"
}
```
5. Нажмите **"Execute"**

**Успешный ответ:**
```json
{
  "id": "796497aa-6ff6-42fd-947e-d561abc5154d",
  "companyName": "ООО Технологии",
  "foundingDate": "2020-01-15T00:00:00Z",
  "accountStatusId": "a6f79508-321e-419b-9998-a1f577f2525c",
  "accountStatusName": "Активный"  // ✅ Заполнено!
}
```

## 🎯 Частые вопросы

### ❓ Почему `accountStatusName` был пустым?

**Причина:** Entity Framework Core не загружал связанную сущность `AccountStatus` автоматически.

**Решение:** Теперь используется явная загрузка (Eager Loading) через `.Include()`:
```csharp
.Include(a => a.AccountStatus)
```

Теперь `accountStatusName` **всегда заполняется** при получении аккаунтов.

---

### ❓ Какие роли могут создавать аккаунты?

- ✅ **Manager** - может создавать и редактировать
- ✅ **Admin** - может создавать, редактировать и удалять
- ❌ **User** - только просмотр

---

### ❓ Как создать статус с уникальным именем?

Система автоматически проверяет уникальность имени статуса. Если вы попытаетесь создать статус с существующим именем, получите ошибку:

```json
{
  "status": 400,
  "title": "Validation Error",
  "errors": {
    "Name": ["Статус с таким именем уже существует"]
  }
}
```

---

### ❓ Можно ли создать аккаунт без статуса?

❌ Нет, поле `accountStatusId` обязательно при создании аккаунта.

**Валидация:**
- `accountStatusId` - обязательное поле
- Должен быть валидный GUID
- Статус с таким ID должен существовать в базе

---

## 📝 Полезные команды

### Получить все аккаунты:
```http
GET /api/accounts
Authorization: Bearer <token>
```

### Получить аккаунты постранично:
```http
GET /api/accounts/paged?pageNumber=1&pageSize=10
Authorization: Bearer <token>
```

### Фильтр аккаунтов по статусу:
```http
GET /api/accounts/by-status/{statusId}
Authorization: Bearer <token>
```

### Обновить аккаунт:
```http
PUT /api/accounts/{id}
Authorization: Bearer <manager_token>
Content-Type: application/json

{
  "id": "796497aa-6ff6-42fd-947e-d561abc5154d",
  "companyName": "ООО Технологии 2.0",
  "foundingDate": "2020-01-15T00:00:00Z",
  "accountStatusId": "новый_ID_статуса"
}
```

### Удалить аккаунт:
```http
DELETE /api/accounts/{id}
Authorization: Bearer <admin_token>
```
